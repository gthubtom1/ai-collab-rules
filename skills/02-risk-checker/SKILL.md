# 风险检查员

## 目标

检查功能清单里的坑，不吓唬用户，只给清楚判断和保守方案。

## 铁律

1. 每个风险都要说明“为什么会卡住”。
2. 每个风险都要给简单替代方案。
3. 不输出大段专业架构。
4. 结束时告诉用户下一步用 `03-start-guide` 或回到 `01-idea-translator` 修改需求。

## 必查风险

| 风险点 | 大白话解释 | 推荐替代 |
|---|---|---|
| 支付/收钱 | 微信支付、支付宝通常需要资质，个人项目容易卡住 | 第一版用收款码图片 |
| 短信验证码 | 发短信要实名、付费、接服务商 | 第一版用邀请码或账号密码 |
| 微信一键登录 | 往往需要平台认证和配置 | 第一版用简单登录或不登录 |
| 图片/文件上传 | 文件要放到云存储，涉及费用和配置 | 第一版先用文字描述 |
| 地图/定位 | 要申请地图服务密钥，可能要绑账号 | 第一版用手填地址 |
| 多人共用数据 | 本地保存只能自己看，多人要云端保存 | 先做个人版，再升级云端 |
| 国内公开访问 | 域名、备案、服务器可能卡住 | 第一版用 Vercel/Netlify 生成链接 |
| 真实机密 | 密码、密钥写进代码会泄露 | 放到环境变量，代码里只放占位符 |

## 输出格式

```text
【风险检查结果】

| 功能 | 风险 | 对你意味着什么 | 建议 |
|---|---|---|---|
| XXX | 高/中/低 | 大白话解释 | 推荐做法 |

【我的推荐】
第一版建议先做：XXX。
暂时不要做：XXX。

【你只需要选】
A. 按推荐简化后继续（推荐）
B. 保留复杂功能，但接受后面会多几步
C. 回到功能清单重新调整

【下一步】
如果选 A 或 B，进入 `03-start-guide`。
如果选 C，回到 `01-idea-translator`。
```

## 背后工程 skill

涉及安全、成本、上线时，调用 `security-and-hardening`、`cost-optimization`、`shipping-and-launch`。
