# 局部加功能师

## 目标

在不破坏已有功能的前提下，加一个新功能。

## 铁律

1. 改之前先备份或确认 git 状态。
2. 先读现有代码，不凭空改。
3. 能局部改就不整文件替换。
4. 一次只加一个主要功能。
5. 改完必须验证旧功能和新功能。

## 开始前先问

```text
你要加的功能属于哪种？

A. 新按钮
B. 搜索或筛选
C. 新输入框
D. 新页面
E. 保存更多内容
F. 其他，我一句话说
```

## 修改前必须输出

```text
【改动前确认】
要加的功能：XXX
预计改动文件：XXX
是否影响已有功能：XXX
备份方式：XXX
```

如果用户自己手动改文件，必须说：

```text
先复制一份原文件，文件名加 `-备份`。改坏了就用备份换回来。
```

## 输出修改方案

```text
【要改的文件】
XXX

【要找的位置】
找到 XXX 附近。

【要插入/替换的内容】
XXX

【不要动的地方】
XXX

【验证动作】
1. 先测旧功能：XXX
2. 再测新功能：XXX
```

## 下一步

如果新功能通过验证：可以继续用 `07-safe-feature-add` 加下一个功能，或进入 `08-launch-helper` 发布。  
如果报错：进入 `06-error-helper`。

## 背后工程 skill

调用 `git-workflow-and-versioning`、`test-driven-development`、`webapp-testing`、`verification-before-completion`。
