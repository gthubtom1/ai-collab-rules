# 收尾一个开发分支

## 概述

用清晰的选项引导开发收尾，并执行所选的工作流。

**核心原则：先验证测试 → 探测环境 → 给出选项 → 执行选择 → 清理。**

**开始时宣告：** "我正在用 finishing-a-development-branch 技能收尾这次工作。"

## 流程

### 第 1 步：验证测试（先于一切）

跑项目测试套件（`npm test` / `pytest` / `go test ./...` / `cargo test` 等）。
- **测试失败** → 停。展示失败、先修。测试没过不许 merge/PR。
- **测试通过** → 继续。

### 第 2 步：探测环境 + 确定基线分支

- 看是普通仓库还是 worktree、是否 detached HEAD（决定给几个选项、怎么清理）。
- 确定从哪个分支分出来的：`git merge-base HEAD main`（或 master），不确定就问用户「这分支是从 main 分出来的吗？」

### 第 3 步：给用户 4 个选项（用大白话）

```
实现完成、测试通过。你想怎么处理？
1. 合并回 <基线分支>（本地 merge）
2. 推送并创建 Pull Request
3. 暂时保留分支（我以后再处理）
4. 丢弃这次工作
选哪个？
```

（detached HEAD 等特殊情况去掉「合并」，给 3 个选项。）

### 第 4 步：执行所选 + 清理

- 按所选执行；**推送/合并到 main、删分支等不可逆动作前，遵守规则 C2（先备份/确认）与 B⑦（推送前机密扫描）**。
- 完成后清理临时分支/worktree（按环境）。

## 本项目集成说明

本 skill 对应规则 B⑦ 部署上线 / B⑧ 收尾：开发完成、要整合或发布时调用；务必先过 verification-before-completion（测试证据）和 B⑦ 机密闸再合并/推送。
