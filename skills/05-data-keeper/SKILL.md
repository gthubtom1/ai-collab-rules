# 数据保管员

## 目标

让用户填写的内容能保存、读取、删除，并知道数据放在哪里。

## 术语解释

- 本地保存：数据存在你当前浏览器里，简单，但换电脑可能看不到。
- 云端保存：数据存在网上的服务里，别人或换电脑也能看到，但要注册和配置。
- 数据库：保存资料的表格仓库。

## 先问保存范围

```text
这些数据需要谁能看到？

A. 只要我这台电脑能看到（推荐第一版）
B. 我换电脑也要看到
C. 朋友或团队也要看到
D. 我不确定，你帮我选
```

## 再问数据类型

```text
保存的主要是什么？

A. 文字和数字（最简单）
B. 日期和状态
C. 图片或文件
D. 用户账号信息
```

## 推荐策略

| 情况 | 推荐 |
|---|---|
| 个人、小数据、无图片 | 本地保存 |
| 多人共用、换电脑也要看 | 云端保存 |
| 图片/文件 | 先评估是否能第一版不做 |
| 用户账号 | 先确认是否真的需要登录 |

## 输出格式

```text
【保存方案】
建议用：XXX。

【为什么】
大白话解释。

【要改哪里】
文件：XXX
位置：XXX

【验证动作】
填一条测试内容 -> 点保存 -> 刷新页面 -> 内容还在。

【下一步】
如果保存成功，进入 `07-safe-feature-add` 加更多功能，或进入 `08-launch-helper` 发布。
```

## 背后工程 skill

涉及数据库时调用 `postgresql` 或相关存储 skill。涉及敏感信息时调用 `security-and-hardening`。完成前调用 `verification-before-completion`。
